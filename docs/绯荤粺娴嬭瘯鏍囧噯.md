# 系统测试标准

## 1. 测试目标

系统测试用于验证在线学习系统在完整业务流程、权限控制、异常处理、性能、安全性和可用性方面是否满足需求。

## 2. 测试范围

1. 用户注册登录。
2. RBAC 权限控制。
3. 课程创建、审核、上架。
4. 学员课程学习。
5. 视频断点续播。
6. 学习进度记录。
7. 章节测验和考试。
8. 错题本。
9. 线上自习室。
10. WebSocket 实时同步。
11. AI 智能助教。
12. 知识图谱推荐。
13. 管理端日志审计。

## 3. 功能测试标准

### 3.1 登录测试

| 测试项             | 预期结果       |
| ------------------ | -------------- |
| 正确账号密码登录   | 返回 Token     |
| 错误密码登录       | 返回密码错误   |
| 禁用账号登录       | 返回账号被禁用 |
| Token 过期访问接口 | 返回未认证     |

### 3.2 权限测试

| 测试项               | 预期结果 |
| -------------------- | -------- |
| 学生访问教师接口     | 返回 403 |
| 教师访问管理端接口   | 返回 403 |
| 管理员访问管理端接口 | 正常访问 |
| 未登录访问受保护接口 | 返回 401 |

### 3.3 课程测试

| 测试项             | 预期结果               |
| ------------------ | ---------------------- |
| 教师创建课程       | 创建成功，状态为 DRAFT |
| 教师提交审核       | 状态变为 PENDING       |
| 管理员审核通过     | 状态变为 PUBLISHED     |
| 学生查看上架课程   | 正常查看               |
| 学生查看未上架课程 | 无法查看               |

### 3.4 学习进度测试

| 测试项               | 预期结果         |
| -------------------- | ---------------- |
| 视频播放后上报进度   | 成功记录         |
| 刷新页面后继续播放   | 从上次位置播放   |
| 短时间上报异常大进度 | 后端拒绝或修正   |
| 完成全部学习条件     | 生成课程完成记录 |

### 3.5 测验测试

| 测试项           | 预期结果         |
| ---------------- | ---------------- |
| 学生开始考试     | 生成 exam_record |
| 提交客观题       | 自动判分         |
| 提交错题         | 自动加入错题本   |
| 超过截止时间提交 | 拒绝提交         |
| 重复提交         | 按规则拒绝或覆盖 |

### 3.6 自习室测试

| 测试项         | 预期结果               |
| -------------- | ---------------------- |
| 用户加入自习室 | 分配座位               |
| 用户状态变化   | 同房间用户实时收到广播 |
| 番茄钟完成     | 生成 room_record       |
| 用户断线重连   | 恢复原状态             |
| 用户超时未重连 | 自动离开房间           |

### 3.7 AI 助教测试

| 测试项       | 预期结果                |
| ------------ | ----------------------- |
| 文本提问     | 返回解答                |
| 上传题目截图 | 返回 OCR 结果和解题步骤 |
| AI 服务超时  | 返回降级提示            |
| 用户频繁调用 | 触发限流                |

## 4. 性能测试标准

1. 普通查询接口平均响应时间小于 300ms。
2. 登录接口平均响应时间小于 500ms。
3. 课程列表接口支持 1000 并发请求。
4. WebSocket 广播延迟小于 500ms。
5. 自习室单房间支持至少 500 人在线。
6. Redis 高频状态写入不得造成 MySQL 压力。
7. AI 服务超时必须可控，不阻塞主系统线程。

---

## 5. 安全测试标准

1. 密码数据库中不可明文查看。
2. 未登录用户不能访问受保护接口。
3. 学生不能访问其他学生学习记录。
4. 学生不能访问未报名课程资源。
5. WebSocket 无 Token 不得连接成功。
6. 文件上传不能上传可执行脚本。
7. 管理员操作必须写入日志。
8. 接口应防止 SQL 注入和 XSS。

---

## 6. 可用性测试标准

1. 网络断开后视频进度可恢复。
2. WebSocket 断开后可自动重连。
3. Redis 异常时课程学习主流程可用。
4. AI 服务异常时系统不崩溃。
5. 文件服务异常时前端给出友好提示。

---

## 7. 验收标准

系统满足以下条件时可认为通过系统测试：

1. 核心功能流程全部通过。
2. 严重安全漏洞为 0。
3. 阻塞级 Bug 为 0。
4. 主要接口响应时间符合要求。
5. 自习室实时同步稳定。
6. AI 服务异常时具备降级能力。
7. 管理端操作具备审计记录。
