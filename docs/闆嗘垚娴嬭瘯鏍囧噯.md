# 集成测试标准

## 1. 测试目标

集成测试用于验证不同模块之间的数据流、权限流、状态流和异常处理是否正确。

## 2. 核心集成链路

## 2.1 登录认证链路

```text
用户登录
  -> AuthController
  -> UserService
  -> PasswordEncoder
  -> JWT Provider
  -> Redis Token Cache
  -> 返回 Token
```

验证点：

1. 正确账号密码可以登录。
2. 禁用用户不能登录。
3. Token 可用于访问受保护接口。
4. Token 过期后访问失败。

------

## 2.2 课程学习链路

```
教师创建课程
  -> 管理员审核
  -> 学生查看课程
  -> 学生报名课程
  -> 学生观看视频
  -> 上报学习进度
  -> 生成学习统计
```

验证点：

1. 未审核课程学生不可见。
2. 上架课程学生可见。
3. 未报名学生不能访问受限资源。
4. 学习进度可正常保存。
5. 断点续播可恢复。

------

## 2.3 测验与错题链路

```
教师创建题目
  -> 教师创建试卷
  -> 学生开始考试
  -> 学生提交答案
  -> 系统自动判分
  -> 错题进入错题本
  -> 知识点薄弱分析
```

验证点：

1. 客观题自动判分正确。
2. 错题自动收录。
3. 成绩记录正确。
4. 用户不能访问他人成绩。
5. 错题知识点可用于推荐。

------

## 2.4 自习室链路

```
学生加入自习室
  -> 后端分配座位
  -> Redis 保存状态
  -> WebSocket 广播
  -> 学生开始番茄钟
  -> 完成专注
  -> MySQL 保存专注流水
```

验证点：

1. 加入房间后 Redis 状态正确。
2. 同房间用户收到广播。
3. 房间满员时不能加入。
4. 番茄钟完成后生成 room_record。
5. 断线重连后状态恢复。

------

## 2.5 AI 助教链路

```
学生上传问题
  -> 主系统鉴权
  -> 主系统限流
  -> 调用 AI 微服务
  -> AI 返回结构化结果
  -> 主系统保存调用日志
  -> 前端展示结果
```

验证点：

1. 未登录不能调用 AI。
2. 图片格式非法时拒绝。
3. AI 服务超时时返回降级提示。
4. AI 调用日志保存成功。
5. 推荐资源不存在时不返回无效资源。

------

## 3. 测试环境要求

1. MySQL 使用独立测试库。
2. Redis 使用独立测试实例或独立 DB。
3. AI 服务可使用 Mock 服务。
4. 文件存储可使用本地 MinIO。
5. 测试数据应可重复初始化。

------

## 4. 数据一致性测试

1. Redis 自习室状态与 MySQL 最终流水一致。
2. 用户提交答案后 exam_record 与 user_answer 一致。
3. 学习进度上报后 video_progress 与 learning_daily_stat 一致。
4. 课程完成后 certificate 与 study_record 一致。
5. 删除课程时不得破坏历史学习记录。

------

## 5. 并发测试

1. 多用户同时加入同一自习室。
2. 多用户同时提交考试。
3. 同一用户多端同时上报视频进度。
4. 多教师同时上传资源。
5. AI 接口高频调用触发限流。

------

## 6. 验收标准

1. 所有核心链路集成测试通过。
2. 权限越权测试全部被拦截。
3. Redis 与 MySQL 数据一致性符合预期。
4. WebSocket 广播稳定。
5. AI 服务失败时系统可降级。
